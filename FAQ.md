# Frequently Asked Questions (FAQ)

## General Questions

### What is this project?
This is a collection of professional portfolio templates designed for developers, designers, and creatives. All templates are free and open-source.

### Are these templates free to use?
Yes! All templates are released under the MIT License, which means you can use them freely for personal or commercial projects.

### Do I need programming knowledge to use these templates?
Basic HTML/CSS knowledge is helpful, but not required. Each template comes with documentation to help you customize it.

## Technical Questions

### Which browsers are supported?
All modern browsers are supported:
- Chrome (latest 2 versions)
- Firefox (latest 2 versions)
- Safari (latest 2 versions)
- Edge (latest 2 versions)

### Are the templates responsive?
Yes! All templates are fully responsive and work on desktop, tablet, and mobile devices.

### Can I use these templates for commercial projects?
Absolutely! The MIT License allows both personal and commercial use.

### Do I need to credit the original authors?
While not required by the MIT License, attribution is always appreciated!

## Customization Questions

### How do I change colors?
Colors are typically defined in the CSS file. Look for color values (hex codes like `#000000`) and replace them with your preferred colors.

### How do I add my own images?
Replace the placeholder images in the template's image folder with your own. Make sure to use the same file names or update the HTML references.

### Can I combine features from different templates?
Yes! Feel free to mix and match features, but you'll need some HTML/CSS knowledge to do this effectively.

### How do I change fonts?
Fonts are usually linked in the HTML `<head>` section or defined in CSS. You can replace Google Fonts links or change the `font-family` CSS property.

## Deployment Questions

### Where can I host my portfolio?
Popular free options include:
- GitHub Pages
- Netlify
- Vercel
- Cloudflare Pages

See our [Deployment Guide](DEPLOYMENT.md) for detailed instructions.

### Do I need a custom domain?
No, you can use free subdomains provided by hosting services. However, a custom domain looks more professional.

### How do I set up a custom domain?
This depends on your hosting provider. Check our [Deployment Guide](DEPLOYMENT.md) for platform-specific instructions.

## Contributing Questions

### How can I contribute?
See our [Contributing Guide](CONTRIBUTING.md) for detailed instructions on how to contribute templates, bug fixes, or improvements.

### I found a bug. What should I do?
Please open an issue on GitHub with details about the bug, including:
- Which template you're using
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable

### Can I submit my own template?
Yes! We welcome new template contributions. Please ensure your template:
- Is responsive
- Works in modern browsers
- Includes documentation
- Follows our coding standards

## Support Questions

### Where can I get help?
1. Check this FAQ
2. Search existing GitHub issues
3. Read the documentation
4. Open a new issue if your question isn't answered

### Is there a community chat or forum?
Currently, we use GitHub Issues and Discussions for community interaction.

### Can I hire someone to customize a template for me?
While we don't provide paid customization services, you can find freelance developers on platforms like Upwork or Fiverr.

## Performance Questions

### How can I optimize my portfolio's loading speed?
- Compress and optimize images
- Minify CSS and JavaScript
- Use a CDN
- Enable caching
- Remove unused code

### Why is my portfolio loading slowly?
Common causes:
- Large, unoptimized images
- Too many external resources
- Slow hosting provider
- No caching configured
